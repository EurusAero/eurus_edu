Работа с API

Работа с API

Eurus-Edu API предоставляет интерфейс управления дроном и периферийными модулями через TCP-соединение с обменом JSON-сообщениями. API предназначено для интеграции с внешними приложениями, образовательными сценариями и автоматизированными системами управления.

Основные возможности API

  • Управление полётом: Запуск моторов, взлёт, посадка, выполнение предзаписанных миссий, управление скоростью и положением.
  • Работа с данными: Получение телеметрии в реальном времени (батарея, координаты, ориентация, скорость), чтение показаний сенсоров (например, лидара).
  • Управление сервисами: Запуск и остановка системных сервисов платформы, проверка их статуса.

Общие сведения:

  • Тип API: TCP Socket API
  • Формат данных: JSON
  • Модель взаимодействия: клиент - сервер - ROS 2
  • Количество клиентов: 1 активный клиент одновременно
  • Асинхронные события: статусы команд, лазертаг

Подключение к серверу

Параметры сервера задаются в конфигурационном файле eurus.ini:

[server]
host = 0.0.0.0
port = 9000
buffer_size = 4096

Подключение клиента

Клиент устанавливает TCP-соединение:

HOST: <ip устройства>
PORT: <port из eurus.ini>

После подключения можно отправлять JSON-сообщения без дополнительной авторизации

Формат сообщений

Общий формат запроса:

{
  "command": "<command_name>",
  "...": "additional parameters"
}

Общий формат ответа:

{
  "command": "action_status | response | response_telemetry",
  "action": "<command_name>",
  "status": "pending | completed | denied",
  "message": "optional text"
}

Статусы команд:

Статус Описание
pending Команда принята и выполняется
completed Команда успешно выполнена
denied Команда отклонена

Управление дроном

Команды управления дроном публикуются в ROS-топик edu/command.

Общие правила:

  • Сервер выполняет только одну команду движения одновременно
  • Если сервер занят, новая команда будет отклонена (denied)

Пример: взлёт

Запрос

{
  "command": "takeoff",
  "altitude": 1.5
}

Ответ:

{
  "command": "action_status",
  "action": "takeoff",
  "status": "pending",
  "message": "Command sent to controller"
}

Ассинхронный ответ (после выполнения):

{
  "command": "action_status",
  "action": "takeoff",
  "status": "completed",
  "message": ""
}

Пример: посадка

{
  "command": "land"
}

Получение телеметрии

Запрос:

{
  "command": "request_telemetry"
}

Ответ:

{
  "command": "response_telemetry",
  "telemetry": {
    "battery": 87,
    "position": { "x": 1.2, "y": 0.0, "z": 1.5 },
    "point_reached": false
  }
}

Проверка достижения точки

Запрос:

{
  "command": "point_reached"
}

Ответ:

{
  "command": "point_reached",
  "point_reached": true
}

Управление LED-лентой

Команда транслируется напрямую в ROS-топик edu/led_control.

Пример:

{
  "command": "led_control",
  "effect": "rainbow",
  "brightness": 0.8
}

Ответ не возвращается (fire-and-forget).

Выстрел лазером

Запрос

{
  "command": "laser_shot"

Ответ:

{
  "command": "action_status",
  "action": "laser_shot",
  "status": "pending",
  "message": "Shot initiated"
}

Поведение при разрыве соединения

При отключении клиента:

  • Сервер автоматически отправляет команду land

  • Активная сессия сбрасывается

  • Новый клиент может подключиться
